# The <code>Tabs</code> Component

## Version 0 - Tabs as props

<TryBadge kind="components-tabs-v0-ðŸš«-tabs-as-props" story="default" />
<DeprecatedBadge />

<div className="subheading">API</div>

```tsx
const content = [
  {label: 'Tab 1', content: 'Content 1'},
  {label: 'Tab 2', content: 'Content 2'},
]
<Tabs tabs={content} />
```

<div className="subheading">Advantages</div>
<ul>
  <li>Quick to implement</li>
  <li>Only one component</li>
  <li>Very simple implementation</li>
</ul>

<div className="subheading">Drawbacks</div>
<ul>
  <li><code>label</code> is prop (cf. <LinkTo kind="components-tabs-v0-ðŸš«-tabs-as-props" story="with-icons">icons as labels</LinkTo>)</li>
  <li>Not customizable at all</li>
</ul>



<hr />

<Tip><p>The following versions have two render methods possible for the content of the tab: conditionally render the content <em>or</em> render all tabs hidden except the current one.
To switch between render methods, check the source code.</p></Tip>

## Version 1 - Labels as props

<TryBadge kind="components-tabs-v1-labels-as-props" story="default" />

<div className="subheading">API</div>

```tsx
<Tabs>
  <Tab label="Tab 1">
    Content 1
  </Tab>
  <Tab label="Tab 2">
    Content 2
  </Tab>
</Tabs>
```

<div className="subheading">Advantages</div>
<ul>
  <li>Only 2 components (<code>Tabs</code> and <code>Tab</code>)</li>
  <li>All information about a tab is at the same place in the code</li>
</ul>

<div className="subheading">Drawbacks</div>
<ul>
  <li><code>label</code> is prop (cf. <LinkTo kind="components-tabs-v1-labels-as-props" story="with-icons">icons as labels</LinkTo>)</li>
  <li>Not very customizable</li>
</ul>

## Version 2 - Sub-components

<TryBadge kind="components-tabs-v2-â­-sub-components" story="default" />

<div className="subheading">API</div>

```tsx
<Tabs>
  <TabList>
    <Tab>Tab 1</Tab>
    <Tab>Tab 2</Tab>
  </TabList>
  <TabContent>
    <div>Content 1</div>
    <div>Content 2</div>
  </TabContent>
</Tabs>
```

<div className="subheading">Advantages</div>
<ul>
  <li>The components follow their natural flow, no need to access the <code>Tabs</code> grand-children to render them separately</li>
  <li>Customizable (relocate tabs in relation to content, inclusion of other components in the midst, <LinkTo kind="components-tabs-v2-â­-sub-components" story="with-icons">label as tabs</a>, ...)</li>
</ul>

<div className="subheading">Drawbacks</div>
<ul>
  <li>Split between label and content</li>
  <li>Loads of components</li>
  <li>More complex implementation (with <code>Context</code>)</li>
</ul>
